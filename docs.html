Documentation

Learn how to install, configure, and use Knobster to control your keyboard wheel.

How it works

Knobster uses keyd, a Linux daemon that remaps keys at the system level. When you select a mode, Knobster writes a config file to /etc/keyd/knobster.conf with the remapping rules and tells keyd to reload.

The volumeup, volumedown, and mute signals from your keyboard get converted into the keys or macros you choose. When you select "Multimedia", the config file is removed and keyd falls back to default volume behavior.

Your /etc/keyd/default.conf is never modified. Your general keyd configuration stays intact.

  Keyboard wheel  ───→  keyd intercepts  ───→  Assigned action
                               ▲
                               │
                       knobster.sh
                      (writes config)

Requirements

  • OS: Linux (tested on Ubuntu 24.04, works on any distro)
  • keyd: Key remapping daemon — github.com/rvaiya/keyd
  • Keyboard: Mechanical keyboard with a volume wheel detectable by keyd (AK820, Keychron, etc.)
  • Terminal: ANSI color support (most modern terminals)
  • Shell: bash 4.0 or higher

Installation

Quick install (1-liner)

curl -fsSL https://raw.githubusercontent.com/epimient/knobster/main/install.sh | bash

This installs keyd if needed, detects your keyboard automatically, and leaves Knobster ready to use.

Note: On Ubuntu/Debian-based distros, the installer compiles keyd from source, which requires git, build-essential, and a few seconds.

Manual installation

1. Install keyd

Ubuntu / Debian:

sudo apt update
sudo apt install -y git build-essential
git clone https://github.com/rvaiya/keyd
cd keyd
make
sudo make install
sudo systemctl enable --now keyd

Arch Linux:

yay -S keyd
sudo systemctl enable --now keyd

Fedora:

sudo dnf install -y git gcc
git clone https://github.com/rvaiya/keyd
cd keyd
make
sudo make install
sudo systemctl enable --now keyd

2. Verify keyd is running

sudo systemctl status keyd

You should see active (running). If not:

sudo systemctl enable --now keyd

3. Download Knobster

git clone https://github.com/epimient/knobster ~/knobster
cd ~/knobster
chmod +x knobster.sh

Configuration

Detect your keyboard ID

Each keyboard has a unique identifier keyd needs to know which device to intercept:

sudo keyd monitor

Turn your keyboard wheel. You'll see something like:

320f:505b:d8fffecf volumeup
320f:505b:d8fffecf volumedown

Copy the first two blocks separated by : — that's your keyboard ID.

Note: The third block (d8fffecf) is a session identifier that changes every time you reconnect the keyboard. keyd only needs the first two blocks.

Compatibility with other keyboards

Knobster works with any keyboard keyd can detect. Not just the AK820. Keychron, Logitech, Razer, Corsair — same process, different ID.

sudo KEYBOARD_ID="04d9:a0a2" ./knobster.sh

Set KEYBOARD_ID

Option 1 — Environment variable:

sudo KEYBOARD_ID="320f:505b" ./knobster.sh

Option 2 — User config file (persistent):

Knobster reads ~/.config/knobster/config if it exists:

KEYBOARD_ID=320f:505b

Option 3 — Edit the script (persistent):

Open knobster.sh and change the KEYBOARD_ID default value.

Customization

Guided setup

From the main menu, select 4 > 1 and follow three steps:

  1. Choose what the wheel does when turned up
  2. Choose what the wheel does when turned down
  3. Choose what happens when you press the wheel

Example keyd commands you can use:

CommandEffect
rightRight arrow
macro(right right right)Right arrow x3
macro(Ctrl right)Ctrl + Right arrow
macro(C-s)Ctrl + S (save)
upUp arrow
pagedownPage down
homeGo to start
escEscape
tabTab
f5F5 key

Direct file editing

From the main menu, select 4 > 2 to open the config file in your editor (nano by default, respects $EDITOR and $VISUAL).

The file is at /etc/keyd/knobster.conf. Save and keyd reloads automatically.

# Knobster
# Modo: Personalizado

[ids]

320f:505b

[main]

volumeup = macro(right right right)
volumedown = macro(left left left)
mute = space

Troubleshooting

keyd not installed

Error: keyd no está instalado.

Follow the Installation guide.

Wheel does nothing after activating a mode

The most common cause is an incorrect keyboard ID.

sudo keyd monitor

Turn the wheel and verify the ID matches your KEYBOARD_ID setting.

keyd reload fails

sudo systemctl status keyd
sudo systemctl enable --now keyd
sudo keyd reload 2>&1

Wheel doesn't work in any app

Possible config syntax error. Clear Knobster's config:

sudo rm -f /etc/keyd/knobster.conf
sudo systemctl restart keyd

Permission denied

Knobster needs root to write to /etc/keyd/.

sudo ./knobster.sh

Menu displays garbled characters

Make sure your terminal supports UTF-8:

export LANG=en_US.UTF-8

Banner not showing

Ensure banner.txt is in the same directory as knobster.sh.

ls -la ~/knobster/banner.txt
